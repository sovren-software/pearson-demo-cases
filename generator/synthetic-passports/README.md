# Synthetic passport dataset generator

A generator for **safe, clearly-synthetic passport test documents** — for validating
LEI's document upload, OCR / data extraction, field mapping, and review flows
**without ever using a real personal document**.

It produces realistic-enough data pages (labelled fields + a valid ICAO-9303 MRZ) to
exercise a real extractor, while remaining **clearly synthetic** (SPECIMEN markings +
fictional data in every JSON sidecar).

There are **two rendering paths**, both safe:

| Path | Tool | Fidelity | Use for |
|---|---|---|---|
| **From-scratch synthetic** | `synth_passports.py` | generic ICAO layout + stylized non-official emblems | any country; the only option for countries with no reusable real reference (e.g. **Canada**, current **UK**) |
| **Adapt real specimen** | `adapt_passports.py` | a real, permissively-licensed passport **specimen/template** with a fictional identity overlaid + SPECIMEN overprint | high-visual-fidelity per-country test docs; bases + licenses catalogued in [`REFERENCES.md`](REFERENCES.md) |

Both stay within the boundary in [`../PASSPORT-SOURCING.md`](../PASSPORT-SOURCING.md):
only government/ICAO specimens, anonymized templates, or CC/PD images, adapted to
**fictional** identities, always marked SPECIMEN. Never a real, identifiable live passport.

---

## Safety posture (by design)

Every generated document:

- carries a red **`SYNTHETIC SPECIMEN · FICTITIOUS TEST DATA · NOT A REAL PASSPORT ·
  NOT VALID FOR TRAVEL OR IDENTITY`** banner across the top, a bottom
  **`SPECIMEN — NOT A REAL DOCUMENT — SYNTHETIC TEST DATA ONLY`** footer, and a
  diagonal **SPECIMEN** watermark;
- uses a **placeholder silhouette** in the photo box — never a real or generated face;
- uses **stylized, geometric, NON-OFFICIAL emblems** on the generic **ICAO-9303 TD3**
  data-page layout (an international standard, not any country's proprietary design) —
  it does **not** replicate an official passport design;
- uses **fictional data only** — names are random combinations from generic common-name
  pools; numbers and dates are synthetic;
- ships a `synthetic: true` flag + disclaimer inside every JSON sidecar.

The goal is *reading/testing our own software*, never producing a usable or
deceptive travel/identity document.

## Quick start

```sh
pip install -r requirements.txt          # Pillow (with raqm) + numpy

python3 synth_passports.py list-countries
python3 synth_passports.py sample                       # curated 120-doc baseline -> out/
python3 synth_passports.py validate --out out           # re-check MRZ + sidecars
```

Generate your own set with controlled variations:

```sh
# 5 docs per country, randomly mixing all quality + defect variants, seeded
python3 synth_passports.py generate --countries all --per-country 5 \
    --quality mix --defects mix --seed 7 --out out

# Just clean Mexico + India docs
python3 synth_passports.py generate --countries MEX,IND --per-country 10 --out out

# A focused defect set (expired + corrupt-MRZ), low-quality scans
python3 synth_passports.py generate --countries all --per-country 2 \
    --quality blur,jpeg,low_res --defects expired,mrz_checkdigit_corrupt --out out
```

Determinism: `sample` and any seeded `generate` are fully reproducible (fixed
`--as-of` reference date, seeded RNG), so the dataset can be regenerated byte-stable.

## Adapt-real path (higher visual fidelity)

`adapt_passports.py` overlays a fictional identity + a fresh valid ICAO MRZ onto a
real, permissively-licensed passport **template**, then marks it SPECIMEN. Bases and
their licenses are catalogued in [`REFERENCES.md`](REFERENCES.md) (verified per file).

```sh
# Proof: adapt the Pakistan blank polycarbonate template (CC BY-SA 4.0) into a
# fictional SPECIMEN. Fetch the base first (see REFERENCES.md for the URL).
python3 adapt_passports.py --reference /path/to/pakistan_poly.png --count 1 --seed 1 --out out_adapt
```

Status: **proof stage** — Pakistan (blank template, no inpainting) is wired end-to-end
with correct ground truth + valid MRZ. The remaining viable countries follow the same
overlay method; bases that carry a real face or redaction boxes (China, Mexico-2016,
India-2021) additionally need LaMa inpainting. **Canada and the current UK passport
have no reusable reference (Crown copyright) → they stay on the from-scratch path.**

Each adapted doc records its base image's `license` + `attribution` + `source_url` in
the JSON sidecar (`base_reference`) — required because several bases are CC BY-SA
(share-alike). Keep that attribution with any redistributed image.

## Countries (the H-1B-weighted set)

`IND · CHN · CAN · KOR · PHL · TWN · MEX · BRA · GBR · PAK` — chosen to mirror the top
H-1B beneficiary countries while spanning **Latin, Devanagari, Han (SC/TC), Hangul, and
Urdu/Perso-Arabic** scripts for OCR-robustness coverage. Edit `countries.py` to add or
adjust styles, name pools, places, and passport-number formats.

### Ground-truth sidecar (`*.json`)

```jsonc
{
  "id": "ind-0004-clean-mrz_visible_mismatch",
  "synthetic": true,
  "disclaimer": "Synthetic specimen for software testing only...",
  "image": "IND/ind-0004-clean-mrz_visible_mismatch.png",
  "country": "IND", "country_display": "India",
  "fields": {                          // the TRUE values a perfect extractor should read
    "surname": "...", "given_names": "...", "nationality": "IND",
    "sex": "M", "date_of_birth": "1990-05-12", "place_of_birth": "MUMBAI",
    "passport_number": "D4518770", "date_of_issue": "...", "date_of_expiry": "...",
    "issuing_authority": "..."
  },
  "mrz": { "line1": "...", "line2": "...", "checkdigits_valid": true },
  "variations": {                      // what was perturbed
    "quality": "clean", "defect": "mrz_visible_mismatch",
    "printed_overrides": { "passport_number": "D9518770" },  // what the IMAGE prints
    "blanked_fields": [], "date_style": "dmy_mon"
  },
  "expected_review_flags": ["mrz_visible_field_mismatch"],   // what LEI review SHOULD surface
  "as_of": "2026-06-25", "seed": "1", "image_sha256": "..."
}
```

`fields` is the extraction oracle. `expected_review_flags` is the **review oracle** —
the integrity findings a correct LEI pass should raise for that document (empty for a
clean doc). `printed_overrides` records where the image was made to diverge from the
true value on purpose (e.g. the printed passport number vs. the MRZ).

## Variation catalog

**Image quality** (`--quality`): how the scan looks.

| variant | effect |
|---|---|
| `clean` | crisp baseline |
| `blur` | gaussian blur (soft scan) |
| `noise` | sensor/scan grain |
| `low_res` | downscale→upscale detail loss |
| `rotate` | small skew with expanded canvas |
| `skew` | mild perspective warp |
| `jpeg` | low-quality JPEG artifacts (saved as `.jpg`) |
| `grayscale` | desaturated photocopy |
| `glare` | bright glare blob |
| `dark` | underexposed |
| `low_contrast` | washed-out |

**Format / defect** (`--defects`): what's wrong with the document.

| variant | `expected_review_flags` |
|---|---|
| `none` | *(clean)* |
| `expired` | `passport_expired` |
| `near_expiry` | `passport_near_expiry` |
| `mrz_checkdigit_corrupt` | `mrz_checkdigit_invalid` |
| `mrz_visible_mismatch` | `mrz_visible_field_mismatch` |
| `missing_field` | `missing_<field>` |
| `date_format_variant` | *(none — parsing-robustness only; renders DD/MM/YYYY, ISO, or DD.MM.YYYY)* |

`--quality` / `--defects` accept `mix` (random per doc), `all`, or a CSV subset.

## MRZ correctness

The bottom MRZ is a real **ICAO-9303 TD3** zone with valid 7-3-1 check digits computed
by the same algorithm as the Pearson corpus (`../generate_cases.py:build_mrz`) and
LEI's `classify_passport_mrz`. Clean docs validate; the `mrz_checkdigit_corrupt` variant
flips the composite check digit so it fails — exercising LEI's MRZ review gate.
`synth_passports.py validate` re-derives every check digit and asserts it matches the
recorded `checkdigits_valid` / `expected_review_flags`.

## Requirements

- **Pillow** built with **raqm** (libraqm) — needed to shape Devanagari conjuncts and
  Urdu RTL. Check with `python3 -c "from PIL import features; print(features.check('raqm'))"`.
- **numpy** — for the `noise` / `skew` variants.
- Fonts: Noto Sans Devanagari, Noto Sans CJK (SC/TC/KR), Noto Naskh Arabic, DejaVu Sans/Mono.

All data is synthetic. See the repo root [README](../../README.md) and
[generator/PASSPORT-SOURCING.md](../PASSPORT-SOURCING.md) for the corpus-wide data policy.
