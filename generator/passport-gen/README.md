# Pearson corpus passport generation (provenance)

How the three demo passports in `cases/h1b/*/passport.png` were produced, for
reproducibility. All data is synthetic; the source images are publicly-available
**specimen** passport samples whose identities were adopted for the demo personas.

- `*-source.png` — the operator-sourced specimen images (China / India / Mexico).
- `*-face.png` — face crops (ImageMagick `-crop` + upscale) used by the renderer.
- `compose.py` — **China**: the source is high-res + clean, so this crops above the
  bottom MRZ band and appends a freshly-computed valid ICAO TD3 MRZ strip onto the
  **real photo page** (native-looking, no field tampering).
- `render_replica.py` — **India + Mexico**: the sources were too low-res / placeholder
  ("SPECIMEN" surname, expired) to edit cleanly, so this renders faithful replicas —
  guilloche security field (procedural spirograph rosettes + wave fields), national
  emblem (Ashoka chakra / seal), the real cropped face + a faint ghost portrait,
  correct bilingual labels (Devanagari / Spanish), CURP, and a valid ICAO MRZ.

The MRZ is computed with the same ICAO 9303 TD3 check-digit algorithm as
`../generate_cases.py` (`build_mrz`), so each passport's MRZ matches the case's
`ground-truth.json` byte-for-byte. The personas + supporting docs are generated by
`../generate_cases.py` (`PROFILES`).

> NOTE: these scripts were authored/run from a scratch dir; their internal paths are
> scratch-relative. They are committed here as provenance + reference; adjust the
> `HERE` / source paths to re-run in place. ImageMagick (`magick`) + the Noto
> Devanagari/CJK + DejaVu/Liberation fonts are required (no PIL on the build host).
